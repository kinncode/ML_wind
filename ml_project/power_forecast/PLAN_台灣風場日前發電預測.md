# 風場風力發電　日前（0–48 小時）預測　專案規劃

> 情境：**單一風場**、**日前 0–48 小時**、資料用**台電/氣象署公開資料 + 你現有的 BSMI 100 m 測風塔**（無風場 SCADA）
> 定位：從「氣象塔研究」延伸到「真實發電預測」，直接對接台灣電網的實際調度需求
> 台灣現況數字皆為 2025–2026 網路查證結果（見文末來源）

---

## 0. 一句話與核心架構

日前風場發電預測，本質上不是「一個模型」，而是**一條四段管線**。單一風場沒有 SCADA 也能做，關鍵是把問題拆成「先預測風、再把風換成電」：

```
  ①  數值天氣預報 (NWP)        ②  偏差校正 / 降尺度         ③  風→電轉換            ④  機率預測
  CWA 全球/區域模式    ──►   用 BSMI 測風塔 + 機器學習   ──►   功率曲線 + 空氣密度   ──►   分位數 p10/p50/p90
  給未來 48h 風速風向        修正 NWP 的系統性誤差(MOS)      + 尾流 + 可用率損失         給調度用的不確定區間
```

**你已經完成的工作，正好是第 ②③ 段的基礎**：測風塔資料處理、風向相依的降尺度模型、空氣密度計算——這些不是重做，是直接接上去。

---

## 1. 為什麼這個題目在台灣有真實價值

台灣正處在再生能源併網的關鍵期，日前風電預測是電網調度的核心痛點：

- **離岸風電裝置容量約 4.8 GW**（2026 年 6 月、逾 500 座風機），目標 2030 年達約 **13 GW**。主要營運風場：**大彰化 900 MW**（沃旭）、**海龍 >1 GW**（2025 年 6 月併網，近七成供台積電）、**允能 640 MW**。
- **台電已建置 48 小時再生能源發電預測系統**來優化調度——代表這正是營運級的真實需求，不是學術練習。
- 風電間歇性對電網慣量與備轉造成壓力，台電正與外部技術合作做日前慣量預測，凸顯「日前尺度」的重要性。
- **學術先例**：2022 年一篇論文用**台灣海峽台電氣象測風塔 2017–2019 資料** + ANN/LSTM 做短期多步風速預測——這與你手上的 BSMI 塔資料幾乎是同一類資產，等於已有可對標的方法。

> 換句話說：這個題目既有**公開的目標資料**（台電發電量）、又有**公開的輸入資料**（氣象署 NWP）、還有**你獨有的地面真值**（BSMI 塔），三者齊備，可行性高。

| 空氣密度修正 | **你已在前處理算好** | 依 IEC 標準修正功率曲線 |

## 3. 為什麼日前預測「一定」要 NWP（用你自己的數據證明）

你之前在 BSMI 塔上算過的 persistence（「下一刻＝現在」）基準線，直接證明了這件事：

| 預測提前量 | Persistence RMSE (m/s) | 對照：猜長期平均 |
|---|---|---|
| 1 小時 | 1.50 | — |
| 6 小時 | 3.40 | — |
| **24 小時** | **5.15** | **5.26（等於失效）** |

**超過約 6 小時，persistence 就逼近「亂猜平均」而失效。** 日前 0–48h 遠在這個範圍之外，所以**單靠塔上觀測外推做不到日前預測，必須引入 NWP 當未來資訊來源**。這也是本規劃把 NWP 放在管線第一段的原因——不是選擇，是物理上的必要。

## 4. 核心方法：四段管線

### ① NWP 取得日前風況
從 CWA 開放資料抓未來 0–48h、對應風場座標與輪轂高度的風速/風向（模式輸出多在標準氣壓層，需內插到輪轂高度約 100–120 m）。

### ② 偏差校正 / 降尺度（MOS）— **你的核心優勢**
NWP 有系統性偏差（地形、海陸、解析度造成）。用 **BSMI 塔的實測值當真值**，訓練一個機器學習模型去修正 NWP：

- 輸入：NWP 預報風速/風向、預報時間、季節/日夜週期、（可選）NWP 的其他變數（氣壓場、溫度）
- 目標：塔實測的輪轂高度風速/風向
- 這就是氣象界的 **MOS（Model Output Statistics）**，而你做過的「風向相依降尺度」正是同一套技術——把 NWP 的粗預報，用地面觀測 + 機器學習修得更準。

### ③ 風 → 電轉換
校正後的輪轂高度風速，透過**功率曲線**換成電功率：
`P = powercurve(U, ρ) × (1 − 尾流損失) × 可用率`
- 空氣密度 ρ 修正：**你已經會算**（冬夏差 5%，直接影響出力）
- 尾流（wake）損失：風機互相遮蔽，整場出力低於單機加總
- 可用率（availability）：維護、故障、切出風速停機

### ④ 機率預測
調度不能只要一個數字，要**不確定區間**。用分位數迴歸（LightGBM `objective=quantile`）直接輸出 p10/p50/p90，畫可靠度曲線。這對電網備轉容量規劃是剛需。

## 5. 單一風場沒有 SCADA，怎麼做得起來？（誠實對策）

你選了「單一風場」但資料只有公開 + BSMI 塔（沒有該風場的真實出力 SCADA）。這是真實世界最常見的處境，有三條可行路徑，建議**同時採用 A + B**：

**A. 合成目標法（虛擬風場出力）— 主力**
用 BSMI 塔（或 NWP 校正後）的風速，套一個代表性機型的功率曲線，算出「假設在此風況下的風場出力」當作訓練與驗證目標。整條管線（①②③④）都能跑，且第 ②段的風速有塔的真值可嚴格驗證。**限制**：最終電量是物理推算而非實測，尾流/可用率是假設值。

**B. 用公開容量因數校準損失**
台電的月風機發電統計（dataset 29961）可算出實際容量因數，用它反推合理的尾流 + 可用率總損失係數，讓合成出力貼近真實水準。

**C. 若目標風場出力恰在台電公開資料中**
先查 dataset 8931 是否有可辨識的風場/機組出力；若有，直接當真實目標，情境立刻升級為「有真值的單場預測」。

> 誠實結論：**「預測風速」這一段有嚴格真值（塔），可以做到有數字、可驗證；「風速→電量」這一段是物理轉換，準不準取決於功率曲線與損失校準。** 報告中要清楚區分這兩段的可信度——這本身就是專業度的展現。

## 6. 模型選型（延續你已驗證的路線）

| 管線段 | 建議模型 | 理由 |
|---|---|---|
| ② 偏差校正 | **LightGBM** 迴歸；輸入 NWP 風速風向 + 塔的近時觀測 lag + 時間週期編碼 | 你已驗證樹模型在此類表格資料上又快又準，CPU 筆電可跑 |
| ④ 機率預測 | LightGBM 分位數迴歸（p10/p50/p90）| 調度需要不確定區間 |
| （進階選配）| 淺層 LSTM / TCN 融合 NWP 序列 | 可做對照，但 baseline 先用樹模型 |

**必備基準線（每個都要跑，證明加值）**：
1. Persistence（證明日前必須 NWP）
2. 氣候平均（該月該時歷史均值）
3. **NWP 直接轉電**（不做偏差校正，直接把 NWP 風速套功率曲線）← 最關鍵對照
4. **NWP + 你的 MOS 校正** ← 主模型，要打敗第 3 項

技術得分：`Skill = 1 − RMSE_模型 / RMSE_NWP直接`，證明「加了塔 + 機器學習校正」真的有用。

## 7. 評估指標（產業標準 + 電網相關）

- **正規化誤差** nRMSE / nMAE ＝ 誤差 ÷ 風場裝置容量（%），這樣不同風場可比較，也是業界慣例
- **技術得分** vs persistence、vs NWP 直接（見上）
- **分位數品質**：pinball loss、可靠度曲線（predicted p90 是否真的涵蓋 90%）
- **調度相關**：日前每小時 MW 誤差、尖峰/離峰分層、爬升事件（ramp）誤差
- **分季節 / 分風速區間**：避免整體平均被冬季高風速主導（你在前一個專案已建立這個好習慣）

## 8. 路線圖（分階段，可漸進交付）

| 階段 | 內容 | 產出 |
|---|---|---|
| 0（0.5 天）| 申請 CWA API Key；下載台電發電開放資料、ERA5 帳號 | 資料到位 |
| 1（1–2 天）| 資料管線：對齊 NWP、塔、發電量的時間軸；建 persistence / 氣候基準 | `data_pipeline.py` |
| 2（2–3 天）| **第 ② 段 MOS 偏差校正**（塔真值）——先把「風速預測」做到有數字 | `mos_correction.ipynb` |
| 3（2 天）| **第 ③ 段 風→電轉換** + 合成目標 + 容量因數校準損失 | `power_curve.py` |
| 4（2 天）| **第 ④ 段 機率預測** + NWP 直接基準對照 | `probabilistic.ipynb` |
| 5（1–2 天）| 回測、分層評估、寫報告、視覺化 | 完整 repo + 報告 |

建議先把第 2 段（風速偏差校正）做扎實，因為那是**唯一有嚴格真值、最能展現你塔資料價值**的一段，也最快看到成果。

## 9. 風險與限制（先寫清楚）

1. **單一風場出力多為 IPP 非公開** → 用合成目標 + 容量因數校準（第 5 節）
2. **NWP 空間解析度** 未必精準對到外海風場位置 → 用塔做 MOS 校正正是為了緩解這點
3. **尾流與可用率** 無 SCADA 難精準校準 → 以公開容量因數約束，並在報告中標為假設
4. **颱風等極端事件** 切出風速停機、NWP 誤差放大 → 單獨分析極端時段
5. **BSMI 塔位置 vs 目標風場位置** 若相距較遠，塔的代表性下降 → 選離塔近的風場，或把塔當「區域風況校正」而非單點真值

## 10. 一頁總結

> 用**台電公開發電資料**當目標、**氣象署 NWP**當未來輸入、**你的 BSMI 測風塔**當地面真值，把日前風場發電預測拆成「NWP → 塔校正 → 功率曲線 → 機率」四段管線。你先前的測風塔處理、風向相依降尺度、空氣密度計算，全部可直接接上第 ②③ 段。單一風場雖無 SCADA，但「預測風速」有嚴格真值可驗證，「風速轉電量」用功率曲線 + 公開容量因數校準——整條管線可行、可交付、且對接台灣電網的真實需求。
